Cloudflare Full-Stack Internship 
Assignment 
Unified Social Media Ads SaaS Platform 
Overview 
This assignment simulates building a real-world Ads Automation SaaS platform similar to 
tools like Unified APIs (e.g., Unified.to, Ayrshare). 
Candidates will build a system that allows users to: 
● Create and manage ad campaigns 
● Publish ads across multiple platforms 
● Track campaign status and performance 
● Manage API integrations centrally 
Supported platforms include: 
● Meta (Facebook / Instagram) 
● Google Ads 
● YouTube 
● TikTok 
● LinkedIn 
● X (Twitter) 
● Snapchat 
● Pinterest 
● Reddit 
● Microsoft Advertising 
● Amazon Ads 
● Quora 
● Spotify 
● WhatsApp 
● Threads 
● Telegram 
Objective 
Build a Unified Ads Management Platform where users can: 
● Connect multiple ad platform accounts 
● Create and publish ads to multiple platforms 
● Manage campaigns centrally 
● Track posting status and logs 
● Extend platform integrations easily 
Core Features 
1. Authentication & User Management 
● Simple login system 
● Token-based authentication 
● Store sessions in KV or D1 
2. Platform Integration Layer (CRITICAL) 
Design a plug-and-play integration system: 
Requirements: 
● Add/remove platforms dynamically 
● Store API credentials securely (KV) 
● Normalize APIs across platforms 
Each platform should support: 
● Authentication (OAuth or API key-based — mock allowed) 
● Post creation (ad creative + metadata) 
● Status tracking 
3. Campaign Management 
Users should be able to: 
● Create campaign: 
○ Title 
○ Budget 
○ Target platforms 
○ Schedule 
● Add creatives: 
○ Image / Video / Text 
● Save draft or publish 
Store in D1 
4. Ad Publishing Engine 
● User selects platforms → publishes campaign 
Flow: 
1. User submits campaign 
2. Create jobs per platform 
3. Push jobs to Queue 
4. Worker processes: 
○ Calls platform APIs (or mocks) 
○ Stores response 
○ Updates status 
5. Media Management 
● Upload creatives: 
○ Images 
○ Videos 
● Store in R2 
● Link media to campaigns 
6. Durable Objects (Stateful Layer) 
Use for: 
● Campaign orchestration 
● Rate limiting per platform 
● Tracking job execution state 
7. Status Tracking Dashboard 
● Show: 
○ Campaign status per platform 
○ Success / Failed / Pending 
○ API responses 
8. Logs & Audit Trail 
Track: 
● API calls 
● Errors 
● Publishing history 
Store in D1 or KV 
9. Caching (KV) 
● Cache: 
○ Platform configs 
○ Access tokens (if short-lived)
